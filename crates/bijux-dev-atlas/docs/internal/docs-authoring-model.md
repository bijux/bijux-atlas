# Internal Docs Authoring Model

This page defines how internal control-plane docs are authored.

## Authoring Rules

- Keep internal authored pages focused on intent, invariants, and workflows.
- Move large inventories, tables, and command dumps to generated outputs.
- Link to generated artifacts instead of duplicating structured data.

## Why/How vs Lists

- Why and how stay in authored markdown under `crates/bijux-dev-atlas/docs/**`.
- Lists and snapshots should be generated by commands and stored in governed generated sinks.
