# Ops Doc Necessity Checklist

- Owner: `bijux-atlas-operations`
- Purpose: `gate new docs and doc expansions so ops documentation surface stays compressed and intentional`
- Consumers: `checks_ops_docs_governance`
- Authority Tier: `machine`
- Audience: `contributors`

## Checklist

- [ ] The document has a single clear consumer (check, generator, runtime workflow, or reviewer flow).
- [ ] The document declares its authority tier and audience.
- [ ] Normative content is backed by machine truth (schema, inventory, or check) when enforceable.
- [ ] The document does not restate tables or indexes that can be generated from inventory.
- [ ] The document is not duplicating semantics already defined in another machine-tier doc.
- [ ] The document explains why deletion would break an identified consumer or workflow.
- [ ] The document is linked from a canonical index or contract.
- [ ] The document can be kept under the applicable doc budget.
