# Ops Directory

This directory is the operations SSOT for runtime workflows.

- `ops/e2e/`: local and CI end-to-end stacks, fixtures, and scripts.
- `ops/load/`: load and performance scenarios, docker compose, and k6 assets.
- `ops/observability/`: dashboards, alerting, and metrics-related operational assets.
- `ops/openapi/`: generated OpenAPI snapshots and versioned specs.

Use this tree as the canonical home for operational artifacts and execution tooling.
