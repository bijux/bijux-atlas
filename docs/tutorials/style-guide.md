# Tutorial Style Guide

Authoring rules:

- Keep each tutorial focused on one outcome with `Goal`, `Steps`, and `Expected result`.
- Prefer generated includes for command lists, schema snippets, and OpenAPI surfaces.
- Keep inline code blocks short; move large output to generated snippets.
- Link to canonical references instead of duplicating API or schema details.
- Keep wording operational and action-oriented.
