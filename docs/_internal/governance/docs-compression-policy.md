# Docs Compression Policy

- Prefer fewer canonical pages with deeper sections over many shallow pages.
- Keep one canonical page per concept and redirect older aliases.
- New pages require a consolidation rationale when similar content already exists.
- Section growth beyond budget requires explicit governance approval.
