## Enforced Rules

- Stable pages must declare owner, review cadence, stability, and a canonical verification marker.
- Generated markdown must include `Generated by` and `Do not edit by hand`.
- Navigation must keep one onboarding funnel and one governance authority entry.
- Runbooks must include `Prereqs`, `Install`, `Verify`, and `Rollback`.
- Redirects must validate through the canonical MkDocs publication path.

## Why These Rules Exist

- Metadata makes ownership and freshness reviewable.
- Generated headers keep the source of truth obvious.
- Navigation rules prevent competing entrypoints.
- Runbook headings make incident work scannable under pressure.
- Redirect and build checks keep published docs honest.
