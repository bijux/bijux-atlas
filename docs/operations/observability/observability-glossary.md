# Observability Glossary

- **Readiness**: whether a node should receive production traffic.
- **Liveliness**: whether a node should remain running.
- **Cardinality**: number of unique label combinations in metrics.
- **Burn rate**: speed at which error budget is consumed.
- **Correlation ID**: stable id tying logs, traces, and requests together.
- **Diagnostics snapshot**: point-in-time runtime state export from debug endpoints.
