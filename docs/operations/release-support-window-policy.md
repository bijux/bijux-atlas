# Release Support Window Policy

Related ops contracts: `OPS-ROOT-023`.

Each release declares a support window with start, cadence, and end-of-support dates.
Support levels define which severities are patched and how quickly fixes are shipped.
Release notes must state overlap windows between consecutive supported versions.
Operators must receive upgrade guidance before a support window closes.
Contracts fail when support-window metadata is missing or inconsistent with release artifacts.
