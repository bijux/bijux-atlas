# Migrating legacy automation

1. Inventory existing helper script behavior.
2. Implement equivalent behavior as a `bijux-dev-atlas` command.
3. Replace docs/workflow references.
4. Remove old script path and add boundary contract coverage.

Canonical migration mapping is tracked in:
`crates/bijux-dev-atlas/docs/migrations/tools-to-dev-atlas.md`.
