# Diagram Style Guide

## Conventions

- Use left-to-right flow for lifecycle and pipeline diagrams.
- Use top-down flow for structural and control-plane diagrams.
- Keep node labels short and noun-oriented.
- Use consistent subsystem naming: ingest, query, ops, release, audit.

## Legend

- Rectangles represent subsystems or processing stages.
- Directed arrows represent control or data movement.
- Grouping by diagram title defines scope; no implicit hidden components.
