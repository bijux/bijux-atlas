# Style Guide

## Voice

- Use direct, technical language.
- Use present tense for behavior and contracts.
- Prefer concrete nouns over abstractions.

## Tone

- Keep statements factual and verifiable.
- Avoid motivational language and marketing claims.

## Formatting

- Use one heading hierarchy per document.
- Use short sections and explicit bullet lists.
- Use monospace for commands, paths, env vars, and identifiers.
