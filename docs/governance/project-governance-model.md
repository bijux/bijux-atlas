# Project Governance Model

## Purpose

This page explains how governance responsibilities are divided across runtime,
operations, documentation, and repository policy.

## Model

- Runtime-facing behavior belongs in product and contract documentation.
- Maintainer automation belongs in the `bijux dev atlas` control plane and its
  governed registries.
- Repository-wide policy belongs in checked-in governance and config sources.
- Generated references summarize current checked-in state and must be
  reproducible from their sources.

## Stability

This model is stable until the repository changes how it assigns decision
authority or validation ownership.
