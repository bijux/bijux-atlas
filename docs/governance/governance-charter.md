# Governance Charter

## Purpose

This charter defines the repository governance principles: honesty, single
source of truth, explicit ownership, and automation that reflects checked-in
policy rather than tribal knowledge.

## Charter

- Keep repository rules in versioned files that can be reviewed and validated.
- Prefer one authoritative source for each governed concern.
- Make ownership explicit for docs, configs, code, and operations artifacts.
- Treat generated outputs as derived evidence, not as hidden authority.
- Keep public behavior, contracts, and install guidance aligned with the
  checked-in runtime and maintainer surfaces.

## Stability

This charter is a stable governance reference. Change it only when the
repository governance model itself changes.
