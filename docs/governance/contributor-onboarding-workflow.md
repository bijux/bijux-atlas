# Contributor Onboarding Workflow

## Purpose

This page gives new contributors the shortest honest path into the repository's
governed workflow.

## Workflow

1. Read [Start Here](../start-here.md) for the documentation map.
2. Install the runtime or maintainer surface that matches your work.
3. Review [Contributor Workflow](../06-development/contributor-workflow.md) and
   [Automation Control Plane](../06-development/automation-control-plane.md).
4. Confirm the source-of-truth file for the area you are changing before you
   edit generated outputs.

## Stability

This workflow is stable guidance for contributor onboarding and should match the
current repository layout and validation surfaces.
