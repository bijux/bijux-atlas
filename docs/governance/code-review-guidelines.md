# Code review guidelines

- Confirm contract and compatibility impact is explicit.
- Confirm tests match behavioral change.
- Confirm docs and operational runbooks are updated.
- Reject hidden side effects and silent behavior changes.
- Require ownership approval for cross-domain changes.
