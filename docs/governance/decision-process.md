# Project decision process

## Decision classes

- Local: one domain, no external contract change
- Cross-domain: multiple domains, possible integration impact
- Contract-affecting: API/schema/runtime contract or compatibility impact

## Required path

1. Open proposal with context, options, and risk analysis.
2. Link impacted contracts and tests.
3. Capture accepted decision in ADR registry for non-local changes.
4. Merge only after required domain approvals.
