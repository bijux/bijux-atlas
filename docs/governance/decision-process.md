# Decision Process

## Purpose

This page describes how Atlas records durable decisions and how repository
changes should be justified when they alter contracts, ownership, or policy.

## Process

1. Identify the authoritative file or registry for the concern being changed.
2. Update the canonical source before updating generated evidence.
3. Record durable design or governance decisions in an ADR when the choice
   affects long-lived repository behavior.
4. Regenerate derived outputs and run the relevant validation commands.

## Stability

This process is stable guidance for repository decisions. Update it only when
the decision workflow itself changes.
