# ADR Registry

## Purpose

This page defines where Architecture Decision Records belong and what they must
capture.

## Registry Rules

- Use ADRs for decisions that change long-lived contracts, governance rules,
  ownership boundaries, or operating assumptions.
- Keep ADRs in the repository area chosen by the maintainers for decision
  records, and link the affected source-of-truth files.
- Prefer one ADR per durable decision so later readers can understand intent
  without reconstructing history from many unrelated commits.

## Stability

This page is the stable governance rule for decision-record usage in this
repository.
